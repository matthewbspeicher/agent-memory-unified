# Signal API — Product Specification

## Overview

Expose the trading engine's signal and opportunity feeds as a paid API. Traders and automated systems subscribe to receive real-time trading signals generated by the 13-agent strategy engine and Bittensor miner consensus.

## Product Positioning

**Tagline:** Real-time trading signals from multi-strategy AI agents.

**Target customers:**
- Retail traders building automated trading bots
- Algo traders wanting signal feeds for backtesting
- Crypto funds looking for alpha sources
- Developers building trading dashboards / newsletters

## Tier Structure

| Tier | Price | Signals/Day | Symbols | Latency | Reasoning | Webhooks |
|------|-------|-------------|---------|---------|-----------|----------|
| **Free** | $0 | 100 | BTC/USD only | 1hr delayed | No | No |
| **Pro** | $49/mo | 10,000 | All | Real-time | No | No |
| **Enterprise** | $199/mo | 100,000 | All | Real-time | Yes | Yes |

### Free Tier Behavior
- Signals are delayed by 1 hour (configurable in future)
- Limited to BTC/USD ticker
- No access to confidence scores or agent metadata
- Rate limited to 100 requests/day
- Cannot be upgraded via self-serve (requires manual approval)

### Pro Tier Behavior
- Real-time signal delivery
- All supported symbols (BTC, ETH, SPY, QQQ, Kalshi markets)
- Confidence score included
- Source agent name included
- Rate limited to 10,000 requests/day

### Enterprise Tier Behavior
- All Pro features
- Full reasoning text from agent
- Webhook delivery (POST to customer URL on new signal)
- Dedicated rate limit: 100,000 requests/day
- Priority support via email

## API Surface

### Authentication

All endpoints require `X-API-Key` header. API keys are generated via the `/keys` endpoint after Stripe payment verification.

### Endpoints

#### `POST /api/v1/signals/keys`

Create a new API key after Stripe verification.

**Request:**
```json
{
  "name": "my-trading-bot",
  "tier": "pro"
}
```

**Response (201):**
```json
{
  "id": "key_abc123",
  "name": "my-trading-bot",
  "key": "amu_sig_sk_live_xxxxxyyyyzzzz",
  "tier": "pro",
  "created_at": "2026-04-14T12:00:00Z"
}
```

#### `GET /api/v1/signals/keys`

List all API keys for the authenticated account.

**Response (200):**
```json
{
  "keys": [
    {
      "id": "key_abc123",
      "name": "my-trading-bot",
      "tier": "pro",
      "created_at": "2026-04-14T12:00:00Z",
      "last_used_at": "2026-04-14T14:30:00Z"
    }
  ]
}
```

#### `DELETE /api/v1/signals/keys/{key_id}`

Revoke an API key.

**Response (204):** No content.

#### `GET /api/v1/signals`

List signals with filtering.

**Query parameters:**
| Param | Type | Description |
|-------|------|-------------|
| `symbol` | string | Filter by ticker (e.g., `BTCUSD`) |
| `signal_type` | string | Filter by signal type |
| `agent` | string | Filter by source agent |
| `min_confidence` | float | Minimum confidence score (0-1) |
| `since` | ISO datetime | Signals after this time |
| `until` | ISO datetime | Signals before this time |
| `limit` | int | Max results (default 100, max 1000) |

**Response (200):**
```json
{
  "signals": [
    {
      "id": "sig_xyz789",
      "source_agent": "momentum_btc_1h",
      "signal_type": "long_entry",
      "symbol": "BTCUSD",
      "direction": "buy",
      "confidence": 0.82,
      "payload": {
        "entry_price": 67420.50,
        "stop_loss": 66100.00,
        "take_profit": 69500.00
      },
      "timestamp": "2026-04-14T14:30:00Z",
      "expires_at": "2026-04-14T15:30:00Z"
    }
  ],
  "meta": {
    "total": 1,
    "limit": 100,
    "has_more": false

}
```

### Webhook Delivery (Enterprise only)

For Enterprise tier, configure a webhook URL to receive signals in real-time.

#### `POST /api/v1/signals/webhooks`

Register webhook URL.

**Request:**
```json
{
  "url": "https://your-server.com/webhooks/signals",
  "events": ["signal.new", "signal.expired"],
  "secret": "your-webhook-secret"
}
```

**Response (201):**
```json
{
  "id": "whk_def456",
  "url": "https://your-server.com/webhooks/signals",
  "events": ["signal.new", "signal.expired"],
  "created_at": "2026-04-14T12:00:00Z"
}
```

#### Webhook payload

```json
{
  "event": "signal.new",
  "timestamp": "2026-04-14T14:30:00Z",
  "data": {
    "id": "sig_xyz789",
    "source_agent": "momentum_btc_1h",
    "signal_type": "long_entry",
    "symbol": "BTCUSD",
    "direction": "buy",
    "confidence": 0.82,
    "payload": {
      "entry_price": 67420.50,
      "stop_loss": 66100.00,
      "take_profit": 69500.00
    }
  }
}
```

Webhook requests include `X-Signal-Signature` header for HMAC verification.

## Data Model

### `signals_api_keys` table (new)

```sql
CREATE TABLE signals_api_keys (
    id TEXT PRIMARY KEY,
    account_id TEXT NOT NULL,
    name TEXT NOT NULL,
    key_hash TEXT NOT NULL UNIQUE,
    tier TEXT NOT NULL DEFAULT 'free',
    stripe_customer_id TEXT,
    stripe_subscription_id TEXT,
    webhook_url TEXT,
    webhook_secret_hash TEXT,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    last_used_at TIMESTAMPTZ,
    revoked_at TIMESTAMPTZ
);
```

### `signals_api_usage` table (new)

```sql
CREATE TABLE signals_api_usage (
    key_id TEXT NOT NULL,
    period_start TIMESTAMPTZ NOT NULL,
    request_count INTEGER NOT NULL DEFAULT 0,
    PRIMARY KEY (key_id, period_start)
);
```

## Stripe Integration

### Products

| Product | Price ID (Stripe) |
|---------|-------------------|
| Signal API Free | N/A |
| Signal API Pro | `price_signal_api_pro` |
| Signal API Enterprise | `price_signal_api_enterprise` |

### Flow

1. User selects tier on landing page
2. Redirect to Stripe Checkout with product price
3. On success, Stripe webhook fires `checkout.session.completed`
4. Server creates API key, sends to user via email
5. On subscription update/cancel, update key tier accordingly

### Webhooks to handle

- `checkout.session.completed` — New subscription
- `customer.subscription.updated` — Tier change
- `customer.subscription.deleted` — Cancellation
- `invoice.payment_failed` — Payment failure notification

## Rate Limiting

| Tier | Requests/Day | Burst |
|------|--------------|-------|
| Free | 100 | 10/min |
| Pro | 10,000 | 500/min |
| Enterprise | 100,000 | 5000/min |

Rate limit headers returned on all responses:
- `X-RateLimit-Limit`
- `X-RateLimit-Remaining`
- `X-RateLimit-Reset`

## Error Responses

| Status | Code | Description |
|--------|------|-------------|
| 401 | `invalid_api_key` | API key missing or invalid |
| 403 | `tier_limit_exceeded` | Rate limit exceeded for tier |
| 404 | `not_found` | Resource not found |
| 422 | `validation_error` | Invalid request parameters |
| 429 | `rate_limited` | Too many requests |
| 500 | `internal_error` | Server error |
