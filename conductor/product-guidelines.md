# Product Guidelines

## Prose Style (Tone & Voice)
- **Visionary & Bold:** Communication should reflect the cutting-edge nature of AI agent memory and autonomous trading. Use strong, active language that emphasizes power, autonomy, and future-forward capabilities.
- **Technical Precision:** While bold, maintain high technical accuracy. Avoid fluff; focus on the impact and efficiency of the system's core capabilities (e.g., "The Redis event bus synchronizes cross-service state in milliseconds" rather than "It's really fast").

## Visual Identity
- **Neural Mesh (Dark Mode):** The primary UI theme is a sophisticated dark mode inspired by neural networks. Use deep blacks, charcoal grays, and high-contrast neon accents (e.g., cyan, electric blue, or emerald green) to highlight critical data and agent activity.
- **Dynamic Density:** The UI should feel "alive" with data while remaining legible. Use high-density layouts for trading charts and memory logs, balanced by strategic whitespace and 3D depth.

## UX Principles
- **Real-time Feedback:** Every agent action, trade execution, and memory storage event must provide immediate visual feedback. Use micro-animations, toast notifications, or live-updating status indicators to ensure the user is always informed.
- **Immersive Visualization:** Prioritize the 3D Knowledge Graph as the primary way to explore agent memories. Use depth and spatial relationships to help users understand complex patterns and connections in their data.
- **Information Density:** Design for power users. Critical trading data and memory logs should be accessible with minimal navigation. Use dashboards that maximize screen real estate for data visibility without overwhelming the user.

## Design Standards
- **Component Consistency:** All UI components (buttons, cards, modals) must follow the "Neural Mesh" design system across all dashboard pages.
- **A11y:** Ensure high color contrast for neon accents against dark backgrounds and maintain full keyboard navigability for critical trading actions.
- **Performance:** The dashboard must be highly responsive, especially when rendering large datasets or 3D visualizations. Optimize rendering pipelines to maintain high frame rates.
