Drawer Behavior — How should it feel?

The drawer is how you go from "glance" to "detail" without leaving Mission Control

A

Overlay Drawer (recommended)

Slides in from the right over the dashboard content. Semi-transparent backdrop dims the panels behind. Click backdrop or press Escape to close. Dashboard stays mounted underneath.

Pros

  • Simple mental model — open/close, dashboard is always "there"
  • Drawer can be wide (50-60% of viewport) for dense content
  • Easy to implement with existing React patterns

Cons

  • Can't see dashboard panels while drawer is open
B

Push Drawer

Slides in from the right and pushes dashboard content to the left. Dashboard panels compress/reflow to fill remaining space. No backdrop needed.

Pros

  • Dashboard panels remain visible (compressed) alongside detail
  • Side-by-side: glance at status bar while reading agent logs

Cons

  • Dashboard panels may get too narrow to be useful
  • More complex layout math (responsive reflow)
  • Can feel janky if not carefully animated
C

Full-Width Takeover

Clicking a panel transitions the entire content area to that panel's detail view. Back button or breadcrumb to return. More like page navigation but within the Mission Control shell.

Pros

  • Maximum space for detail content
  • Simplest to build — just conditional rendering
  • No drawer animation complexity

Cons

  • Loses the "single pane of glass" feel
  • Feels more like page navigation than drill-down
